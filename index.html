Heading Level 1

The <h1> element also known as "Main Heading" represents the highest level of heading in a document.

It serves as the main heading of a page and should represent the primary topic. It should be used only once per page to indicate the main focus. Search engines give significant weight to this tag, which helps them understand the main topic of the page. Screen readers use it as a starting point for the content. Typically, it is used for the main title or heading of a webpage or a major section. To maintain proper document structure, there should usually be only one per page.

  • Primary Title: Represents the main title of a webpage or a major section within a page, indicating the primary topic or purpose.
  • SEO Importance: Highly influential for search engine optimization (SEO), as search engines use to understand the main topic of the page. Generally, only one should be used per page for best practices.
Heading Level 2

The <h2> element also known as "Section Heading" represents the second level of headings.

It marks major sections within the page and can be used multiple times to divide content into significant sections. It helps search engines understand the structure and sub-topics of the content and provides a way for users with screen readers to navigate between sections. It is used for subheadings or sections under the main title. Headings help to divide content into major sections that fall under the main topic introduced by the main heading.

  • Secondary Titles: Used for subheadings under the main title, breaking down content into major sections that support the main topic.
  • Content Organization: Helps to organize content into distinct sections, making it easier for users to navigate and for search engines to index.
Heading Level 3

The <h3> element also known as "Subsection Headings" is the third level of headings.

It helps break down the content into more digestible parts. It further divides sections into smaller, more detailed parts. It can be used multiple times within sections, adding additional structure and assisting search engines in understanding more granular topics. It also facilitates detailed navigation within sections for screen reader users.

  • Detailed Breakdown: Used to create subsections under headings, allowing for more detailed and specific content organization.
  • Enhanced Readability: Improves readability by further dividing content into manageable parts, making it easier for readers to follow.
Heading Level 4

The <h4> element also known as "Detailed Headings" is the fourth level of headings, used for smaller subdivisions within sections.

This is useful for creating a hierarchy within a major section. It adds another level of detail within subsections. It is used when there is a need to further subdivide content. Although it has less impact than higher-level tags, it is still valuable for detailed content structuring. It contributes to a finely-grained navigation structure. It helps to further organize content into smaller, more manageable parts.

  • Granular Detail: Provides another level of content subdivision within sections, useful for detailing complex topics.
  • Content Hierarchy: Maintains a clear content hierarchy, helping both readers and search engines understand the structure and importance of information.
Heading Level 5

The <h5>element also known as "Nested Headings" represents the fifth level of headings.

It helps to detail specific points or aspects within a subsection. It provides even more granular division within detailed sections. It is used sparingly and only when necessary to further subdivide content. While it has minimal impact on SEO, it is useful for detailed documents and enhances the navigational structure for users utilizing assistive technologies. It is used for even finer subdivisions within sections.

  • Finer Subdivision: Used for even more granular subdivisions within sections, useful for very detailed content.
  • Supporting Information: Ideal for adding additional layers of supporting information without overwhelming the primary content.
Heading Level 6

The <h6> element also known as "Lowest Level Headings" represents the lowest level of headings in HTML.

This is useful for highly detailed and complex documents. It is used for the most specific and detailed headings and is rarely used except when absolutely necessary to divide content further. It has the least impact on SEO but still provides some structure. It is used for the smallest subdivisions within a document's structure.

  • Lowest Level: Used for the most granular level of content organization.
  • Rarely Used: Often not necessary in most documents, as five levels of headings are usually sufficient.
Best Practices

Properly structured headings are vital for creating accessible, SEO-friendly, and user-friendly web pages. By following the hierarchical order and best practices outlined below, you can ensure that your content is well-organized and easy to navigate for both users and search engines.

When using HTML headings, consider the following best practices:

  • Use headings in order (H1, then H2, etc.) without skipping levels.
  • Use only one H1 per page for the main title.
  • Keep headings concise and descriptive.
  • Use headings to create a logical document outline.
